# 註解方針

#### 單行註解 \(Single Line Comments\)

> **堅持** 以好的命名與架構表達語意，而非以大量註解說明。
>
> **考慮** 使用註解說明演算法內容、商務邏輯、設計考量等。

```csharp
// 用於說明程式碼
// 避免大量註解反而造成閱讀困難
```

> **考慮** 使用 `HACK`、`TODO`、`UNDONE` 等語彙基元建立工作清單。

```csharp
// HACK: 重構方法
// TODO: 效能調教
// UNDONE: 參數檢核
```

> **禁止** 使用排版符號。

```csharp
// ----------
// 禁止使用
// ----------
```

#### 分隔註解 \(Delimited Comments\)

不使用。

```csharp
/* 不使用此種註解方式 */
```
